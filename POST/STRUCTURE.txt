# Перенос программы на другой компьютер (без репозитория)

Все файлы в этой папке сохранены с расширением **.txt**, чтобы их можно было пересылать или хранить без потери содержимого. На целевом компьютере каждый файл нужно **переименовать** (убрать .txt в конце) и положить по указанному ниже пути.

---

## Структура папки проекта на целевом компьютере

Создайте папку проекта, например:

```
JSON_to_CSV_XLSX_Universal_Parser/
├── main.py              ← из main.py.txt
├── config.json          ← из config.json.txt
├── IN/                  ← создать пустую папку, положить сюда JSON-файлы
├── OUT/                 ← создаётся программой при первом запуске
├── log/                 ← создаётся программой при первом запуске
└── src/
    ├── __init__.py      ← из src/__init__.py.txt
    ├── config_loader.py ← из src/config_loader.py.txt
    ├── csv_exporter.py  ← из src/csv_exporter.py.txt
    ├── json_flattener.py← из src/json_flattener.py.txt
    ├── logging_setup.py ← из src/logging_setup.py.txt
    ├── worker.py        ← из src/worker.py.txt
    ├── xlsx_exporter.py ← из src/xlsx_exporter.py.txt
    └── Tests/
        ├── __init__.py  ← из src/Tests/__init__.py.txt
        └── test_flattener.py ← из src/Tests/test_flattener.py.txt
```

## Порядок действий на целевом компьютере

1. Создать папку проекта (например `JSON_to_CSV_XLSX_Universal_Parser`).
2. Создать внутри неё папку **src**, а внутри src — папку **Tests**.
3. Создать папку **IN** (для входных JSON-файлов).
4. Каждый файл из POST переименовать: убрать **.txt** в конце и сохранить по пути из таблицы выше (main.py и config.json — в корень проекта, остальные .py — в src/ или src/Tests/).
5. Открыть **config.json** и при необходимости изменить:
   - **input_dir** — папка с JSON (по умолчанию `IN`);
   - **output_dir** — папка для результатов (по умолчанию `OUT`);
   - **files** — массив с полем **file**: имена JSON-файлов из папки IN.
6. Положить входные JSON-файлы в папку IN (имена должны совпадать с указанными в config.json в files[].file).
7. Из **корня проекта** выполнить в терминале:
   ```bash
   python main.py
   ```
8. Результаты появятся в папке OUT: CSV-файлы и один XLSX `output_YYYYMMDD-HHMM.xlsx`. Логи — в папке log.

---

## Требования на целевом компьютере

- **Python 3.7+** (стандартная поставка с python.org или Anaconda). Дополнительные пакеты (pip install) **не требуются** — программа использует только стандартную библиотеку.

---

## Проверка (тесты)

Из корня проекта:

```bash
python src/Tests/test_flattener.py
```

Если вынесете проект в другую структуру каталогов, в начале файла `src/Tests/test_flattener.py` может понадобиться поправить путь в `sys.path.insert`.
